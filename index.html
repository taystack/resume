Bread and butter?

Easily HTML5, JavaScript, and CSS3. I create, contribute to, and maintain web applications with them. These tools have allowed me to breathe life into creative designs from creative designers. I'm a stickler for standards and always refer to the World Wide Web Consortium (W3C) for standards and conventional practices in web development. That said, I am not opposed to a last minute hack before a deployment schedule. Or Easter Eggs. Grinning for either.

HTML can be very nice to use when cleaned up with an abstraction layer or other sweet templating engine. I spend most of my time inside Haml templates but have come to appreciate other markup languages like, Jade, and Jinja which also allow a variety of logic from the application architecture's native language. This website was created with Haml using grunt-haml for compilation.

I have coded in javascript or some other abstraction layer more than any other coding / scripting language. Learning JavaScript (es5) came to me by way of learning CoffeeScript and LiveScript. My first raw JavaScript web application was written well after my involvement with multiple production-level CoffeeScript / LiveScript web apps. The most recent applications I have worked on are utilizing the bleeding-edge JavaScript (es6) syntax. This specific website was written using plain ol' JavaScript (es5).

I heart this stuff. This is where the lines merge between art and math, and the fastest way for a browser to understand what the heck it's supposed to show you. With JavaScript, HTML5, and CSS3, I can make just about anything you can think of. This website was created with Sass using grunt-contrib-sass for compilation. In fact, if your browser doesn't support flex, this whole site looks broken!
